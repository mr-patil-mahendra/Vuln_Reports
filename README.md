# Vulnerability Reports Repository

## Overview

This repository contains documented security vulnerability reports identified during security research, lab practice, bug bounty activities, and responsible disclosure assessments.

The purpose of this repository is to maintain organized, structured, and reproducible security findings for learning, documentation, and reporting purposes.

---

## Repository Structure

Each vulnerability report typically includes:

- Vulnerability Title
- Severity Level
- Affected Endpoint / Component
- Vulnerability Description
- Impact Assessment
- Steps to Reproduce
- Proof of Concept (PoC)
- Supporting Evidence / Screenshots
- Remediation Recommendations
