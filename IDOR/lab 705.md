## Title
 Insecure Direct Object Reference (IDOR) at SecureBank 
 
## Vulnerability Type
Insecure Direct Object Reference (IDOR)

## Summary  
The application in http://kzlabs.in/705.php is vulnerable to Insecure Direct Object Reference (IDOR), allowing an attacker to manipulate object identifiers (such as user IDs or record IDs) to access resources belonging to other users without proper authorization checks. This vulnerability can result in unauthorized access to sensitive information and actions on behalf of other users.

## Vulnerable Endpoint
http://kzlabs.in/705.php

## Steps to Reproduce  
1. Navigate to the following URL : `http://kzlabs.in/705.php`
2. Log in to the application with valid Account1.
3. Navigate to the Support Center section.
4. Observe the following URL: `http://kzlabs.in/705.php?action=ticket&id=2`
5. Change the value of the `id` parameter from: `2` to `4` 
6. You got access to Support Center of another Users

## Impact
 - Unauthorized Access to Sensitive Data
 - Unauthorized Modification of Data
 - Data Breach and Privacy Violations
   
## Recommendations for fix
 - Implement Server-Side Authorization Checks
 - Enforce Access Control on Every Request
 - Validate User Ownership
