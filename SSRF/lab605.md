## Title
 Server-Side Request Forgery (SSRF) at SecureAuth

## Vulnerability Type
Server-Side Request Forgery (SSRF)

## Summary  
In http://kzlabs.in/605.php.  website fetches the URL from user input which is Vulnerable to SSRF. which allows attacker to make the server send requests to internal systems or external services, leading to information disclosure or further attacks .

## Vulnerable Endpoint
http://kzlabs.in/605.php

## Steps to Reproduce  
1. Navigate to the following URL : `http://kzlabs.in/605.php`
2. Go to Campaign Section.
3. In `Campaign: Summer Sale 2024`, You will see the Input box for `Add and Scam Link`
4. As i enter the   `File:///etc/passwd/` You see the server returns the Sensitive file’s contents.  


## Payload Used
`File:///etc/passwd`

## Proof of Concept Request
  ```Screemshot 1 : ``` TThe Report Shows Payload add in Input Box. Location where payload should be insert. 




## Impact
 - It allows attackers to access and scan internal systems, mapping infrastructure behind the firewall.
 - It allows attackers to read sensitive local files (e.g., /etc/passwd) and retrieve cloud metadata to steal credentials.
 - It allows attackers to interact with internal APIs and services, potentially leading to remote code execution.
  
## Recommendations for fix

 - Use CSRF tokens
 - Use SameSite Cookie Attributes
 - Validate Origin and Referer Headers
 - Avoid State Changes Through GET Requests
