## Title
 Insecure Direct Object Reference (IDOR) at Medicare+

## Vulnerability Type
Insecure Direct Object Reference (IDOR)

## Summary  
The application in http://kzlabs.in/703.php is vulnerable to Insecure Direct Object Reference (IDOR), allowing an attacker to manipulate object identifiers (such as user IDs or record IDs) to access resources belonging to other users without proper authorization checks. This vulnerability can result in unauthorized access to sensitive information and actions on behalf of other users.

## Vulnerable Endpoint
http://kzlabs.in/703.php

## Steps to Reproduce  
1. Navigate to the following URL : `http://kzlabs.in/703.php`
2. Log in to the application with valid Account1.
3. Navigate to the Appoinments Section.
4. Click on View Details.
5. Observe the following URL: `http://kzlabs.in/703.php?action=appointment&id=3`,  same for  Lab Results section and Prescription Section
6. Change the value of the `id` parameter from: `3` to `6` 
7. You got access to Appoointment of another Users

## Impact
 - Unauthorized Access to Sensitive Data
 - Unauthorized Modification of Data
 - Data Breach and Privacy Violations
   
## Recommendations for fix
 - Implement Server-Side Authorization Checks
 - Enforce Access Control on Every Request
 - Validate User Ownership
